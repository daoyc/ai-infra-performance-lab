# Week 001

## 本轮目标

建立 AI Infra 转岗前的**初始技能基线**，把当前已验证能力、部分验证能力和后续发展方向明确分层。

## 评估输入

- 简历基线：`道一川.pdf`
- 追问材料：问答 `Q1-Q6`
- 补充材料：`AI-agent+GLM项目.txt`

## 本轮确认的高可信技能

- Flink 分布式性能优化
- `perf / Topdown / FlameGraph` 驱动的问题定位
- NUMA 绑核与跨 P / 跨 die 访存优化
- 鲲鹏 920 场景性能验证与资源规划
- Shell 自动化性能采集与归档

## 本轮部分验证技能

- CPU 微架构理解深度
- C/C++ 性能敏感代码分析能力
- Linux 内核 / 虚拟内存原理
- AI-agent 工作流工程化

这些能力已经有可讨论的具体案例，但还需要更多独立实践，才能进一步升级结论强度。

## 关键证据结论

- 对 `L3 miss`、`branch mispredict`、`IPC` 偏低但 CPU 利用率较高等现象，能够从缓存争用、流水线 flush、指令依赖和 Topdown 角度给出较完整的解释。
- Nexmark 场景中，对象拷贝路径与 `pipeline.Reuse` 参数优化带来了端到端性能收益。
- 鲲鹏 920 小规格场景中，`NUMA` 资源绑定、I/O 与调度策略组合带来了可观收益。
- `AI-agent + GLM` 相关工作证明了工程流程设计、工具化与知识沉淀能力，但当前更适合作为次核心能力，而不是主线标签。

## 为什么 AI-agent 记为次核心标签

当前最稳定、最有连续证据的主线仍然是性能优化与系统调优。  
`AI-agent` 相关经历更能体现流程设计、协作效率和工具链抽象能力，因此放在次核心位置更准确，也更能避免和未完全补齐证据的 AI Infra / LLM 推理能力混写。

## 下一步补证据建议

- 为 `CPU 微架构指标分析` 补充更多不同类型瓶颈案例。
- 为 `C/C++ 性能敏感代码分析能力` 补充更直接的代码级优化记录。
- 为 `AI Infra / LLM 推理优化` 建立独立 benchmark、报告与代码证据。
- 后续每次新增实践时，显式标注它支撑了哪个技能升级。
