# 技能基线

本文件用于记录当前技能状态、证据来源、能力边界与后续升级条件。

## 等级说明

- `了解`：知道核心概念，但缺乏稳定复述或实践。
- `能复述`：能较清楚解释原理、指标与常见问题。
- `能实践`：做过具体场景，能说明工具、动作和结果。
- `能主导`：不仅能实践，还能拆解问题、组织协作并沉淀方法。

## 状态说明

- `已验证`
- `部分验证`
- `发展中/待补证据`

## 本轮高可信证据

- 能围绕 `L3 miss`、`branch mispredict`、`IPC` 偏低等典型现象，从缓存争用、分支预测失败、流水线 flush 与指令依赖角度解释微架构层面的潜在瓶颈。
- Nexmark 场景中，通过 `pipeline.Reuse` 与对象拷贝路径优化，使端到端任务性能提升约 8%。
- 鲲鹏 920 小规格场景中，通过 `NUMA` 绑核、资源规划、I/O 优化和调度策略组合获得收益。
- AI-agent + GLM 场景中，通过增量编译、CLI 工作流、Markdown memory 与 Wiki 沉淀降低团队重复劳动。

## 技能升级规则

- 只有同时具备“具体场景 + 使用工具 + 个人动作 + 指标变化”，才允许升级技能等级。
- 只有“理解概念”，不能升级到 `能实践`。
- 只有“参与项目”，不能自动升级到 `能主导`。
- 后续任何 `log / benchmark / report` 都需要注明它支撑了哪个技能升级。
